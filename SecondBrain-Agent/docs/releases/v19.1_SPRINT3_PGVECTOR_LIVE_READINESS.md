# v19.1 Sprint 3 — PostgreSQL/pgvector Live Readiness

## Ziel

Die pgvector Foundation kann jetzt optional eine echte PostgreSQL-Verbindung prüfen und das Schema anwenden.

Der lokale Default bleibt offline-safe: Ohne `--live` und ohne `--apply` wird keine Datenbankverbindung geöffnet.

## Geändert

```text
secondbrain/p3_pgvector_foundation.py
launcher.py
```

## Neue Funktionen

```python
pgvector_live_check(config)
apply_pgvector_schema(config, dry_run=True)
pgvector_readiness(project_root, write_report=False, live=False, apply=False)
```

## CLI

Preview/offline:

```powershell
python launcher.py p3-pgvector-readiness --write-report
```

Live-Check:

```powershell
python launcher.py p3-pgvector-readiness --live --write-report
```

Schema anwenden:

```powershell
python launcher.py p3-pgvector-readiness --apply --write-report
```

`--apply` ist bewusst explizit. Ohne `--apply` wird keine Migration ausgeführt.

## Voraussetzungen für Live-Check

```powershell
pip install psycopg[binary]
$env:SECONDBRAIN_PGVECTOR_ENABLED="true"
$env:SECONDBRAIN_PGVECTOR_DSN="postgresql://appuser:password@localhost:5432/secondbrain"
```

## Live-Check prüft

- psycopg importierbar
- DSN vorhanden
- PostgreSQL-Verbindung erreichbar
- `select version()` ausführbar
- pgvector Extension-Status über `pg_extension`
- DSN im Report redacted

## Apply-Runner

`--apply` führt aus:

```sql
create extension if not exists vector;
create schema if not exists secondbrain;
create table if not exists secondbrain.p1_documents (...);
create table if not exists secondbrain.p1_chunks (...);
create table if not exists secondbrain.p1_chunk_embeddings (... embedding vector(1536) ...);
create index if not exists ...;
```
