# v18.12 Sprint 1 — CI/Reproducibility Hardening

## Ziel

P0 Build/Reproducibility stabilisieren und verhindern, dass Packaging-, Requirements-, README- oder CI-Struktur erneut auseinanderlaufen.

## Änderungen

- GitHub Actions P0-Smoke-Workflow ergänzt: `.github/workflows/secondbrain-ci.yml`.
- RepoDoctor um CI-Prüfung erweitert.
- RepoDoctor um optionale PyProject-Extras erweitert: `dev`, `pdf`, `connectors`, `openai`, `all`.
- RepoDoctor prüft jetzt Projektversion in `pyproject.toml`.
- RepoDoctor prüft Dev-Requirements auf `pytest`.
- RepoDoctor prüft `docs/releases/` als Release-Dokumentationsquelle.
- README auf `docs/releases/` als Source of Truth umgestellt.
- README dokumentiert Root-Artefakte `PATCH_`, `CHANGELOG_`, `VALIDATION_` als unzulässig.
