# v18.14 Sprint 2 — P1 Golden Retrieval Dataset

## Ziel

RAG-Qualität soll nicht mehr nur über hart codierte Probe-Queries geschätzt werden. Dieses Paket ergänzt ein kuratiertes Golden-Retrieval-Dataset und einen eigenständigen Evaluationslauf.

## Neue Komponente

```text
secondbrain/p1_golden_retrieval.py
```

Funktionen:

- Built-in Golden Dataset
- optionales Projekt-Dataset unter `config/golden_retrieval.json`
- Dataset-Validierung
- Hybrid-Search-Evaluation gegen erwartete Begriffe
- Metriken pro Query:
  - `recall_at_k`
  - `mrr`
  - `ndcg`
  - `hit_count`
  - `confidence`
- maschinenlesbarer Report

## Neuer CLI-Befehl

```powershell
python launcher.py p1-golden-eval
python launcher.py p1-golden-eval --write-report
```

Report:

```text
runtime/reports/p1_golden_retrieval_latest.json
```

## Built-in Dataset

Wenn keine Konfiguration existiert, nutzt der Evaluator ein kleines eingebautes Dataset:

```text
Jarvis RAG Quellen
Memory Evidenz
lokale Quellen
```

Das ist kein Ersatz für ein großes fachliches Dataset, aber ein reproduzierbarer Regression-Startpunkt.

## Optionales Projekt-Dataset

Datei:

```text
config/golden_retrieval.json
```

Beispiel:

```json
{
  "dataset_id": "project_v1",
  "queries": [
    {
      "query": "Jarvis RAG Quellen",
      "expected_terms": ["jarvis", "rag", "quellen"],
      "min_recall_at_k": 0.66,
      "min_mrr": 0.25,
      "min_ndcg": 0.35,
      "k": 10,
      "note": "Core source retrieval smoke"
    }
  ]
}
```
