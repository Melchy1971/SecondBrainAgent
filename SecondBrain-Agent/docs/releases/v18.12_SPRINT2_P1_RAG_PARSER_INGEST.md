# v18.12 Sprint 2 — P1 RAG Parser-Integrated File Ingest

## Ziel

`p1-rag-ingest-file` nutzt nicht mehr blind `Path.read_text()`, sondern die bestehende Document-Understanding-Parser-Schicht.

Damit wird der P1-RAG-Import anschlussfähig für:

- Markdown
- Plain Text
- JSON/JSONL
- CSV
- EML
- PDF-Textparser mit explizitem OCR/Fehlerstatus

## Änderungen

### Runtime

Geändert:

```text
secondbrain/p1_rag_runtime.py
```

Neue Verdrahtung:

```text
P1RagRuntime
  -> default_parser_registry()
  -> ParsedDocument
  -> ingest_text(..., metadata=parse_metadata)
```

### Fehlerpolitik

`ingest_file()` blockiert nun sauber bei:

| Parserstatus | RAG-Fehler |
|---|---|
| `unsupported` | `unsupported_file_type` |
| `failed` | `parse_failed` |
| `ocr_required` | `ocr_required` |
| `empty` | `empty_text` |

Diese Fehler erzeugen keine kaputten Dokumente im RAG-Index.

### Metadaten

RAG-Dokumente erhalten Parser-Metadaten:

```json
{
  "path": "...",
  "ingest_mode": "parser_registry",
  "mime_type": "...",
  "parse_status": "parsed",
  "parser": "...",
  "chars": 123,
  "pages": 0
}
```

Bei Parserfehlern wird zusätzlich `parse_errors` gespeichert.
