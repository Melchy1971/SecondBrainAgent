# v18.15 Sprint 2 — P1 Production Golden Gate

## Ziel

`p1-production` soll nicht mehr nur technische Retrieval-Metriken prüfen, sondern zusätzlich das Golden Retrieval Dataset bestehen müssen.

## Neue Komponente

```text
secondbrain/p1_production_gate.py
```

Funktion:

```python
production_gate_with_golden(runtime, project_root, write_report=False)
```

Diese Funktion kombiniert:

- bestehendes `runtime.production_gate()`
- `evaluate_golden_retrieval(...)`
- gemeinsamen Production-Report

## Neuer Production-Report

`p1-production` erzeugt jetzt:

```json
{
  "schema": "secondbrain.p1_production.golden.v1",
  "base_production": {},
  "golden_retrieval": {},
  "checks": [
    {
      "name": "golden_retrieval_eval_passes",
      "ok": true,
      "severity": "blocker"
    }
  ]
}
```

Reportpfad bleibt kompatibel:

```text
runtime/reports/p1_production_latest.json
```

## CLI-Verhalten

```powershell
python launcher.py p1-production --write-report
```

läuft jetzt über:

```text
production_gate_with_golden(...)
```

## Gate-Politik

Production ist nur PASS, wenn:

- das bisherige P1 Production Gate besteht
- Embedding Provider production-ready ist
- kein Fallback-Embedding verwendet wird
- Golden Retrieval Eval besteht
- Quellen-/Antwortpolitik eingehalten wird

## Tests
