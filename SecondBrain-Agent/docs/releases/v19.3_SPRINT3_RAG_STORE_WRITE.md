# v19.3 Sprint 3 — RAG Store Write Interface

## Ziel

Das RAG Store Interface wird von read-only auf einen klaren Schreibvertrag erweitert.

SQLite kann jetzt Dokumente, Chunks und Vektoren direkt schreiben. pgvector liefert SQL-Preview für dieselben Operationen und echte Similarity-Search-SQL mit `<=>`.

## Erweiterter Store-Vertrag

```python
upsert_document(document)
upsert_chunks(chunks)
upsert_vectors(vectors)
vector_search(query_vector, limit=5, provider=None)
```

## SQLiteRagStore

Jetzt implementiert:

- Schema-Initialisierung
- `upsert_document`
- `upsert_chunks`
- `upsert_vectors`
- `sources`
- `vector_search`
- `status`

Damit kann ein RAG-Index über den Store aufgebaut werden, ohne direkt SQL in Runtime-Code zu schreiben.

## PgVectorRagStore

Foundation-only, aber mit echten SQL-Vorlagen:

- `build_upsert_document_sql()`
- `build_upsert_chunk_sql()`
- `build_upsert_vector_sql()`
- `build_vector_search_sql(provider=None)`

Die pgvector Search nutzt:

```sql
1 - (e.embedding <=> %s::vector) as score
order by e.embedding <=> %s::vector
```

Live-Ausführung ist noch nicht aktiv. Dadurch bleibt der Schritt sicher und testbar.
