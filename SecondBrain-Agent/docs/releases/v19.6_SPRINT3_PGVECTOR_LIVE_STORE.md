# v19.6 Sprint 3 — pgvector Live Store Execution

## Ziel

`PgVectorRagStore` ist nicht mehr nur SQL-Preview. Der Store kann jetzt live gegen PostgreSQL/pgvector ausführen, wenn `SECONDBRAIN_PGVECTOR_ENABLED=true` und eine DSN gesetzt ist.

## Wichtig

Secrets werden nicht im Repo gespeichert. Die DSN kommt ausschließlich lokal aus Environment-Variablen, zum Beispiel:

```powershell
$env:SECONDBRAIN_PGVECTOR_ENABLED="true"
$env:DATABASE_URL="postgresql://appuser:***@host:5432/secondbrain_prod"
```

Reports zeigen die DSN nur redacted.

## Geändert

```text
secondbrain/p3_rag_store.py
```

Schema:

```text
secondbrain.p3_rag_store.v3
```

## Live-Funktionen

`PgVectorRagStore` unterstützt jetzt:

- `status()` mit optionalen Live-Counts
- `sources()` live
- `upsert_document()` live
- `upsert_chunks()` live
- `upsert_vectors()` live
- `vector_search()` live mit pgvector cosine distance

## pgvector Search

Die Suche nutzt weiter:

```sql
1 - (e.embedding <=> %s::vector) as score
order by e.embedding <=> %s::vector
```

## Fehlerverhalten

Wenn `psycopg` fehlt:

```json
{
  "ok": false,
  "status": "blocked",
  "error": "psycopg_missing"
}
```

Wenn DSN fehlt:

```json
{
  "ok": false,
  "status": "blocked",
  "error": "dsn_missing"
}
```

Wenn pgvector deaktiviert ist:

```json
{
  "ok": false,
  "status": "blocked",
  "error": "pgvector_disabled"
}
```

## Lokale Validierung ohne echte DB

Die Tests verwenden Fake-`psycopg` und benötigen keine echte Datenbank.

```powershell
pytest -q tests/test_v191_p3_rag_store.py
```

## Live-Validierung mit VPS-DB

```powershell
cd H:\SecondBrainAgent\SecondBrain-Agent
pip install psycopg[binary]

$env:SECONDBRAIN_PGVECTOR_ENABLED="true"
$env:DATABASE_URL="postgresql://appuser:***@85.215.131.200:5432/secondbrain_prod"

python launcher.py p3-pgvector-readiness --live --write-report
python launcher.py p3-pgvector-readiness --apply --write-report
python launcher.py p3-rag-store-status
```

## Was jetzt möglich ist

Nach `--apply` kann der Store live Dokumente, Chunks und Vektoren schreiben sowie Vector Search ausführen.

## Noch offen

- `P1RagRuntime` wählt noch nicht automatisch `PgVectorRagStore` für Live-Ingest.
- SQLite -> PostgreSQL Migration Runner fehlt noch.
- P1-Suchpfade sind noch nicht vollständig store-backed.
