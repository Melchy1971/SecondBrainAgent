# v30.0 Provider Layer Delta

## Ziel
Einheitliche Provider-Schicht für Chat, Embeddings, Streaming-Iteration, Routing, Metriken, Rate-Limits, Failover und Kostenabschätzung.

## Neue Struktur
- `secondbrain/providers/base/*`
- `secondbrain/providers/openai/chat_provider.py`
- `secondbrain/providers/anthropic/chat_provider.py`
- `secondbrain/providers/gemini/chat_provider.py`
- `secondbrain/providers/ollama/chat_provider.py`
- `secondbrain/providers/routing/*`

## Integrationslogik
Bestehende RAG-Provider bleiben kompatibel. Die neue Schicht ist eine produktionsfähigere Abstraktion darüber und kann schrittweise an Launcher, RAG und Agent Runtime angebunden werden.

## Konfiguration
Umgebungsvariablen:
- `OPENAI_API_KEY`
- `ANTHROPIC_API_KEY`
- `GEMINI_API_KEY`

Ollama nutzt standardmäßig:
- `http://localhost:11434`

## Risiken / offene Punkte
- Echtes SSE-Streaming ist als Iterator-Vertrag vorbereitet, aber stdlib-basiert noch nicht als Tokenstream umgesetzt.
- Anthropic-Embeddings sind nicht unterstützt.
- Provider-Auswahl ist noch nicht zentral im Launcher verdrahtet.
