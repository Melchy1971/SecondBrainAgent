# v19.2 Sprint 3 — RAG Store Interface

## Ziel

Der RAG-Code bekommt eine klare Store-Abstraktion, damit SQLite und PostgreSQL/pgvector künftig parallel unterstützt werden können.

Dieser Schnitt ist bewusst read-only/foundation:

- SQLite-Adapter liest den bestehenden `runtime/p1_rag/rag.sqlite3`
- pgvector-Adapter zeigt Konfiguration/SQL-Preview
- pgvector `sources()` und `vector_search()` sind explizit `not_implemented`
- keine versteckten Live-DB-Operationen

## Neue Komponente

```text
secondbrain/p3_rag_store.py
```

Enthält:

- `RagStore` Protocol
- `RagDocumentRecord`
- `RagChunkRecord`
- `RagVectorHit`
- `SQLiteRagStore`
- `PgVectorRagStore`
- `create_rag_store(project_root)`
- `store_backend_from_config(project_root)`

## Neuer CLI-Befehl

```powershell
python launcher.py p3-rag-store-status
```

Zeigt den aktiven Store an:

```text
sqlite
pgvector
```

## SQLite-Adapter

Liest bestehende Tabellen:

- `documents`
- `chunks`
- `chunk_embeddings`

Funktionen:

- `status()`
- `sources()`
- `vector_search(query_vector, limit=5, provider=None)`

## pgvector-Adapter

Aktuell Foundation-only:

- `status()` zeigt Konfiguration und SQL-Preview
- `sources()` meldet `pgvector_sources_not_implemented_yet`
- `vector_search()` meldet `pgvector_vector_search_not_implemented_yet`

Damit ist klar sichtbar, was bereits bereit ist und was noch nicht.
