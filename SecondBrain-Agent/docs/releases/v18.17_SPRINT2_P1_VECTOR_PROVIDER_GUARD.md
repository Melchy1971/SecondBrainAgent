# v18.17 Sprint 2 — P1 Vector Provider Reindex Guard

## Ziel

Wenn der Embedding Provider gewechselt wird, dürfen alte Vektoren nicht still weiter als produktionsreif gelten.

Beispiel:

```text
local-deterministic -> openai
local-deterministic -> ollama
openai -> ollama
```

In diesen Fällen muss der VectorStore neu aufgebaut werden.

## Neue Komponente

```text
secondbrain/p1_vector_provider_guard.py
```

Neuer Audit:

```python
audit_vector_provider(runtime, write_report=False)
```

Prüft:

- aktueller aktiver Provider
- vorhandene Embedding-Provider im SQLite VectorStore
- stale vectors mit anderem Provider
- fehlende Vektoren
- Remediation-Hinweis

## Neuer CLI-Befehl

```powershell
python launcher.py p1-vector-provider-audit
python launcher.py p1-vector-provider-audit --write-report
```

Report:

```text
runtime/reports/p1_vector_provider_guard_latest.json
```

## Production Gate

`p1-production` prüft jetzt zusätzlich:

```text
vector_provider_audit_passes
```

Wenn stale Vektoren vorhanden sind, wird Production blockiert.

## Remediation

Nach Providerwechsel:

```powershell
python launcher.py p1-rag-reindex --write-report
python launcher.py p1-vector-provider-audit --write-report
python launcher.py p1-production --write-report
```

## Tests

Neu:

```text
tests/test_v187_p1_vector_provider_guard.py
```

Abdeckung:

- Audit PASS bei aktuellem Provider
- Audit BLOCKED nach Providerwechsel
- Reindex repariert stale Vektoren
- Production Gate enthält Vector Provider Audit
- CLI-Befehl funktioniert
- Command-Index enthält `p1-vector-provider-audit`

## Bedeutung

Dieser Guard verhindert, dass eine Installation scheinbar auf OpenAI/Ollama läuft, während der Index noch mit lokalen deterministic Embeddings erzeugt wurde.
