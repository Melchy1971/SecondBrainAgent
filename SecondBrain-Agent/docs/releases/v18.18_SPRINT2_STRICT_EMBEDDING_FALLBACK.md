# v18.18 Sprint 2 — Strict Embedding Fallback Policy

## Ziel

RAG darf bei konfigurierten Netzwerk-Embedding-Providern nicht still auf lokale Testvektoren zurückfallen. Ein Gate darf nicht PASS melden, während tatsächlich `local-deterministic` statt OpenAI/Ollama verwendet wird.

## Änderung

### Vorher

- `OpenAIEmbeddingProvider.embed()` fing Providerfehler ab.
- `OllamaEmbeddingProvider.embed()` fing Providerfehler ab.
- Beide Provider gaben dann lokale deterministische Embeddings zurück.
- `status()` meldete zwar `fallback_used=True`, aber der eigentliche Ingest-/Reindex-Pfad lief weiter.

### Nachher

- OpenAI/Ollama blockieren bei Providerfehlern mit RuntimeError.
- Lokaler Fallback ist nur noch explizit erlaubt über:

```powershell
$env:SECONDBRAIN_EMBEDDING_ALLOW_FALLBACK="true"
```

- `status()` liefert zusätzlich:

```json
{
  "fallback_allowed": false,
  "fallback_used": false,
  "production_ready": false
}
```

bei blockierten Netzwerkprovidern.

## Akzeptanz

Neue/angepasste Tests:

```text
test_openai_embedding_provider_requires_api_key
test_openai_embedding_provider_reports_missing_sdk
test_openai_embedding_provider_blocks_on_sdk_failure_by_default
test_openai_embedding_provider_fallback_requires_explicit_opt_in
```

## Produktwirkung

- Kein Fake-PASS durch stille lokale Embeddings.
- Providerfehler werden im produktiven RAG-Pfad sichtbar.
- Lokaler Offline-Modus bleibt verfügbar, aber nur als bewusst gesetzter Entwicklungsmodus.
