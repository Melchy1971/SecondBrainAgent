# v18.19 Sprint 2 — Provider Health Guard

## Ziel

RAG-Vektoroperationen dürfen bei konfigurierten, aber nicht verfügbaren Netzwerk-Embedding-Providern keine Teilzustände erzeugen.

## Änderung

`P1RagRuntime` prüft vor Vektoroperationen den aktiven Providerstatus:

```text
ingest_text
reindex_vectors
vector_search
```

Wenn der Provider ein Netzwerkprovider ist, `status.ok=false` meldet und kein expliziter Fallback erlaubt ist, wird blockiert:

```json
{
  "ok": false,
  "status": "blocked",
  "error": "embedding_provider_unhealthy",
  "operation": "ingest_text"
}
```

## Fallback-Policy

Expliziter Entwicklungsfallback bleibt möglich:

```powershell
$env:SECONDBRAIN_EMBEDDING_ALLOW_FALLBACK="true"
```

Ohne diese Variable blockieren OpenAI/Ollama bei Fehlern.

## Produktwirkung

- Keine stillen Fake-Vektoren bei OpenAI/Ollama-Ausfall.
- Keine halben Ingests bei defektem Provider.
- Reindex erzeugt keine gemischten oder falsch markierten Vektoren.
- Vector Search liefert kontrollierten Blockstatus statt Exception/False Confidence.
