# v19.0 Sprint 3 — PostgreSQL/pgvector Foundation

## Ziel

Sprint 3 startet den Weg vom lokalen SQLite-Prototyp zu produktionsfähiger PostgreSQL/pgvector-Persistenz.

Dieser erste Schnitt ist bewusst eine Foundation:

- keine harte PostgreSQL-Laufzeitabhängigkeit
- kein Live-DB-Zwang in lokalen Tests
- reproduzierbare Config-/Readiness-/SQL-Schicht
- Grundlage für spätere Migration/Live-Verbindung

## Neue Komponente

```text
secondbrain/p3_pgvector_foundation.py
```

Funktionen:

- `load_pgvector_config(project_root)`
- `redact_dsn(dsn)`
- `build_pgvector_schema_sql(config)`
- `pgvector_readiness(project_root, write_report=False)`

## Neuer CLI-Befehl

```powershell
python launcher.py p3-pgvector-readiness
python launcher.py p3-pgvector-readiness --write-report
```

Report:

```text
runtime/reports/p3_pgvector_readiness_latest.json
```

## Konfiguration

### Environment

```powershell
$env:SECONDBRAIN_PGVECTOR_ENABLED="true"
$env:SECONDBRAIN_PGVECTOR_DSN="postgresql://appuser:password@localhost:5432/secondbrain"
$env:SECONDBRAIN_PGVECTOR_SCHEMA="secondbrain"
$env:SECONDBRAIN_PGVECTOR_TABLE_PREFIX="p1"
$env:SECONDBRAIN_PGVECTOR_DIMENSIONS="1536"
```

Alternativ wird `DATABASE_URL` als DSN akzeptiert.

### YAML

Datei:

```text
config/pgvector.yaml
```

Beispiel:

```yaml
enabled: true
dsn: postgresql://appuser:password@localhost:5432/secondbrain
schema_name: secondbrain
table_prefix: p1
vector_dimensions: 1536
provider: openai
```

## DSN-Sicherheit

DSNs werden in Reports redacted:

```text
postgresql://appuser:***@localhost:5432/secondbrain
```

## SQL Foundation

Der Readiness-Check erzeugt ein SQL-Preview für:

- `create extension if not exists vector`
- Schema `secondbrain`
- Tabelle `p1_documents`
- Tabelle `p1_chunks`
- Tabelle `p1_chunk_embeddings`
- `vector(dimensions)` Spalte
- Basis-Indizes inklusive IVFFLAT

## Gate-Verhalten

Wenn pgvector deaktiviert ist und keine DSN existiert:

```text
ok = true
warnings >= 1
```

Wenn pgvector aktiviert ist, aber DSN fehlt:

```text
ok = false
status = blocked
```
