# v19.5 Sprint 3 — P1 Store-backed Ingest

## Ziel

`P1RagRuntime.ingest_text()` schreibt nicht mehr direkt alle Dokumente, Chunks und Embeddings per Inline-SQL, sondern nutzt jetzt das neue RAG Store Interface.

Dieser Schnitt ist bewusst begrenzt:

- Ingest-Schreibpfad ist store-backed
- Reindex-Schreibpfad ist store-backed
- Such-/Gate-Lesepfade bleiben vorerst auf dem bestehenden SQLite-Lesepfad
- pgvector Live-Ingest wird noch nicht aktiviert

## Geändert

```text
secondbrain/p1_rag_runtime.py
```

Neue interne Verwendung:

```python
RagDocumentRecord
RagChunkRecord
RagVectorRecord
SQLiteRagStore
```

## Neuer Ingest-Pfad

`ingest_text()` baut jetzt Store-Records:

```python
RagDocumentRecord(...)
RagChunkRecord(...)
RagVectorRecord(...)
```

und schreibt über:

```python
self.rag_store.upsert_document(document)
self.rag_store.upsert_chunks(chunk_records)
self.rag_store.upsert_vectors(vector_records)
```

Das Ergebnis enthält jetzt:

```json
{
  "store": {
    "backend": "sqlite",
    "write_path": "rag_store"
  }
}
```

## Reindex

`reindex_vectors()` schreibt neue Vektoren ebenfalls über:

```python
self.rag_store.upsert_vectors(...)
```

## Warum nur Ingest zuerst?

Die Runtime enthält noch viele bestehende SQLite-Lesepfade für:

- Keyword Search
- Vector Search
- Sources
- Validation
- Gates

Diese werden nicht im selben Schritt umgebaut, um Regressionen zu vermeiden.
