# v19.4 Sprint 3 — P1 Store Bridge

## Ziel

`P1RagRuntime` wird nicht sofort komplett umgebaut. Stattdessen gibt es jetzt eine sichere Bridge vom bestehenden P1-SQLite-Index zum neuen RAG Store Interface.

Das reduziert Risiko, weil die große Runtime-Datei nicht in einem Schritt ersetzt werden muss.

## Neue Komponente

```text
secondbrain/p3_p1_store_bridge.py
```

Funktionen:

- `export_p1_sqlite_records(db_path)`
- `mirror_p1_sqlite_to_store(source_db_path, target_store)`
- `mirror_project_p1_to_selected_store(project_root, write_report=False)`

## Neuer CLI-Befehl

```powershell
python launcher.py p3-p1-store-bridge
python launcher.py p3-p1-store-bridge --write-report
```

Report:

```text
runtime/reports/p3_p1_store_bridge_latest.json
```

## Verhalten

Wenn der aktive Store weiterhin die bestehende P1-SQLite-Datei ist:

```text
status = noop
```

Wenn ein anderer Store ausgewählt wird, werden vorhandene P1-Daten über den Store-Vertrag gespiegelt:

- Dokumente → `RagDocumentRecord`
- Chunks → `RagChunkRecord`
- Embeddings → `RagVectorRecord`

## Warum Bridge statt Big Bang?

Die bisherige P1-Runtime ist groß und enthält Ingest, Chunking, Embeddings, Suche, Gates und Reports. Ein kompletter Sofort-Umbau auf Store-backed Runtime wäre riskant.

Die Bridge schafft den sicheren Zwischenschritt:

1. bestehende P1-Daten korrekt ins Store-Format exportieren
2. Store-Schreibvertrag validieren
3. Migration/Spiegelung ermöglichen
4. danach P1RagRuntime Schritt für Schritt intern auf Store umstellen
