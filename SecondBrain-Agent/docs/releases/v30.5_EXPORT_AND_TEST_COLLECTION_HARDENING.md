# v30.5 Export- und Test-Collection-Hardening

## Ziel

Der aktuelle Stand scheitert vor der fachlichen Validierung an Import-/Exportgrenzen und Test-Collection-Drift. Dieses Delta stabilisiert die öffentlichen Package-APIs und entfernt die unmittelbare Blockade aus dem Test-Startpfad.

## Änderungen

- Public API Exports für `secondbrain.agent` ergänzt.
- Public API Exports für `secondbrain.desktop` ergänzt.
- Public API Exports für Dashboard, Documents und Search ergänzt.
- Backward-kompatibles `WorkspaceRef.id` Alias ergänzt.
- Legacy-Workspace-Listenverhalten isoliert auf den aktiven Workspace, damit alte Tests nicht durch persistierte `.config`-Daten verunreinigt werden.
- Test-Verzeichnisse als Packages markiert, damit gleichnamige Testdateien in unterschiedlichen Unterordnern keine Import-Mismatch-Fehler erzeugen.

pytest -q -x

## Konsequenz

Der nächste technische Blocker liegt nicht mehr in den fehlenden Public Exports, sondern in vollständiger lokaler Suite-Laufzeit, Runtime-Artefakt-Hygiene und produktiver pgvector-/Connector-Validierung.
