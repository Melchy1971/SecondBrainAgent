### 19. Assistent

Sehr gute Frage – die Größe von PST-Dateien hängt von der Outlook-Version ab:  

- **Alte Outlook-Versionen (bis 2002):** max. ca. **2 GB** → diese alten PSTs sind im „ANSI-Format“.  
- **Moderne Outlook-Versionen (ab 2003, inkl. Outlook 365):** PST-Dateien sind im **Unicode-Format** und können standardmäßig bis zu **50 GB** groß werden.  

### Details für Outlook 365 (Desktop-App)  
- **Standardlimit:** 50 GB pro PST-Datei.  
- **Technisches Maximum:** Microsoft hat intern höhere Grenzen (bis zu 4 TB), aber praktisch ist das **nicht sinnvoll** – Outlook wird ab ~20–30 GB merklich langsamer.  
- **Anpassung möglich:** In der Windows-Registry kann man das Limit hochsetzen, aber Microsoft empfiehlt ausdrücklich, es bei 50 GB zu belassen.  

### Praxis-Tipp  
- Lege lieber **mehrere PST-Dateien** an (z. B. Archiv pro Jahr oder Projekt), statt eine Riesen-PST zu pflegen.  
- Achte darauf, die Datei regelmäßig zu komprimieren: Rechtsklick auf die PST → „Datendateieigenschaften“ → „Jetzt komprimieren“.

Eine **EML-Datei** ist im Grunde eine einzelne, gespeicherte E-Mail. Stell dir vor, du nimmst eine Mail aus deinem Posteingang und packst sie als Datei auf deine Festplatte – genau das ist EML.  

### Eigenschaften einer EML-Datei  
- **Format:** Textbasiert, nach dem Standard **MIME RFC 822** (ein Internetprotokoll für E-Mails).  
- **Inhalt:** Enthält den kompletten Mailtext, Anhänge (meist als codierte Blöcke), Absender, Empfänger, Betreff, Zeitstempel und Header-Infos.  
- **Endung:** `.eml`  

### Wozu nutzt man EML-Dateien?  
- **Archivierung:** Einzelne Nachrichten können unabhängig von einem Mailprogramm gespeichert werden.  
- **Weitergabe:** Du kannst eine EML-Datei wie ein Dokument verschicken, und der Empfänger kann sie wieder in ein Mailprogramm ziehen.  
- **Öffnen:** Viele Programme erkennen EML: Outlook, Thunderbird, Windows Mail, Apple Mail – oder sogar ein Texteditor, wenn man neugierig ist, wie die „Rohdaten“ einer Mail aussehen.  

### Unterschied zu PST  
- **EML:** eine einzelne Mail.  
- **PST:** ein ganzer Container, in dem viele Mails, Ordner, Kontakte etc. stecken können.

### 23. Assistent

Eine **MBOX-Datei** ist so etwas wie die große Schwester der EML-Datei: statt nur einer einzelnen Nachricht speichert sie **eine ganze Sammlung von E-Mails in einer einzigen Textdatei**.  

### Eigenschaften einer MBOX-Datei  
- **Format:** Ebenfalls textbasiert, uralt und weit verbreitet (seit den 1970ern).  
- **Inhalt:** Alle Nachrichten werden hintereinander in einer Datei gespeichert, jeweils durch eine Zeile beginnend mit `From ` getrennt.  
- **Endung:** `.mbox` oder manchmal einfach ohne Endung.  

### Wozu wird MBOX genutzt?  
- **Archivierung:** Viele E-Mail-Programme nutzen MBOX, um komplette Ordner (z. B. „Posteingang 2023“) zu speichern.  
- **Portabilität:** Einfach auf einen Stick ziehen und in ein anderes Programm importieren.  
- **Kompatibilität:** Beliebt bei Open-Source-Clients wie **Mozilla Thunderbird**, **Apple Mail**, oder bei Unix-Systemen mit klassischen Mailservern.  

### Unterschied zu EML und PST  
- **EML:** eine Mail pro Datei.  
- **MBOX:** viele Mails in einer Datei, schlicht hintereinandergehängt.  
- **PST (Outlook):** ein proprietärer Container von Microsoft, der neben Mails auch Kalender, Kontakte, Aufgaben usw. enthalten kann.

## 🔄 Update-Ablauf für ein Open-Source-Mailarchivierungs-Tool

### 1. Vorbereitung
- **Version checken:** Mit `piler --version` (oder entsprechendem Befehl des Tools) prüfen, welche Version läuft.  
- **Release Notes lesen:** Auf GitHub oder der Projektseite schauen, ob spezielle Migrationsschritte nötig sind.  
- **Backup machen:**  
  - Datenbank (oft MySQL/MariaDB)  
    ```bash
    mysqldump -u root -p piler > piler_backup.sql
    ```
  - Konfiguration (meist `/etc/piler/` oder `/usr/local/etc/piler/`)  
  - Archivdaten (Mailstore, oft `/var/piler/`)  

---

### 2. Update einspielen
- **Paketmanager (falls Repo genutzt):**  
  ```bash
  sudo apt update
  sudo apt upgrade piler
  ```
  oder auf RHEL/CentOS:  
  ```bash
  sudo yum update piler
  ```

- **Quellcode (falls aus Source gebaut):**  
  - Neue Version von GitHub holen:  
    ```bash
    wget https://bitbucket.org/jsuto/piler/downloads/piler-<VERSION>.tar.gz
    tar xvf piler-<VERSION>.tar.gz
    cd piler-<VERSION>
    ./configure
    make
    sudo make install
    ```
  - Alte Configs vergleichen und ggf. anpassen (z. B. `piler.conf`).  

---

### 3. Nacharbeiten
- **Datenbank-Migrationen ausführen** (oft mitgeliefert im `db-migrations`-Ordner).  
- **Dienste neu starten:**  
  ```bash
  sudo systemctl restart piler
  ```
- **Logfiles prüfen:** in `/var/log/syslog` oder `/var/log/piler/`.  

---

### 4. Testen
- Web-GUI öffnen → prüfen, ob Login & Suche funktionieren.  
- Test-Mail archivieren → sicherstellen, dass sie korrekt indexiert wird.  

---

💡 Viele Admins nutzen für Mailarchiver bewusst **getrennte Testsysteme**, um Updates erst dort durchzuspielen – das ist gerade bei Langzeitarchiven sehr ratsam.
