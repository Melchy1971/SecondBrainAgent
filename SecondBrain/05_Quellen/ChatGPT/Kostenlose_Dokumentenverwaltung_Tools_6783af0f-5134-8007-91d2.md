Es gibt mehrere kostenlose Programme, mit denen Sie Ihre Dokumente verwalten können. Hier sind einige der besten Optionen:

### 1. **Google Drive**
   - **Funktionen**: Speichern, organisieren und teilen Sie Dokumente in der Cloud. Enthält Google Docs, Google Sheets und Google Slides für die Bearbeitung.
   - **Vorteile**: Kostenloser Speicherplatz (15 GB), einfache Nutzung, plattformübergreifend.

### 2. **Microsoft OneDrive (kostenlose Version)**
   - **Funktionen**: Cloud-Speicherung und Verwaltung von Dokumenten. Integration mit Microsoft Office Online.
   - **Vorteile**: 5 GB kostenloser Speicher, nahtlose Integration mit Windows.
   - **Geeignet für**: Windows-Nutzer, die ihre Dokumente einfach organisieren möchten.

### 3. **OnlyOffice Personal**
   - **Funktionen**: Verwalten und bearbeiten Sie Dokumente direkt in einer kostenlosen Cloud-Umgebung.
   - **Vorteile**: Unterstützung für verschiedene Dokumentenformate, Open-Source-Technologie.
   - **Geeignet für**: Personen, die ein vollwertiges Office-Tool mit Dokumentenverwaltung suchen.

### 4. **LibreOffice**
   - **Funktionen**: Ein Open-Source-Büropaket für die Bearbeitung und Organisation von Dokumenten lokal auf Ihrem Computer.
   - **Vorteile**: Vollständig kostenlos, keine Internetverbindung erforderlich.
   - **Geeignet für**: Nutzer, die eine Alternative zu Microsoft Office suchen.

### 5. **Notion**
   - **Funktionen**: Erstellen, organisieren und verwalten Sie Dokumente, Notizen und Projekte in einer anpassbaren Umgebung.
   - **Vorteile**: Flexibilität, Cloud-basierte Lösung, für Einzelpersonen kostenlos.
   - **Geeignet für**: Personen, die eine Kombination aus Dokumentenverwaltung und Projektmanagement wünschen.

### 6. **Evernote (kostenlose Version)**
   - **Funktionen**: Dokumente speichern, sortieren und Notizen erstellen.
   - **Vorteile**: Synchronisation zwischen verschiedenen Geräten.
   - **Geeignet für**: Nutzer, die hauptsächlich Texte und PDF-Dokumente verwalten.

### 7. **DocFetcher**
   - **Funktionen**: Ein Desktop-Tool zum Durchsuchen und Organisieren von Dokumenten auf Ihrem PC.
   - **Vorteile**: Schnelle Volltextsuche, leichtgewichtig.
   - **Geeignet für**: Nutzer, die lokale Dateien effizient durchsuchen möchten.

### 8. **Zoho Docs**
   - **Funktionen**: Cloud-Speicherung, gemeinsame Dokumentenbearbeitung und Organisation.
   - **Vorteile**: Plattformübergreifend, benutzerfreundlich.

### 8. Assistent

Die Installation von **Paperless-ngx** (eine verbesserte und beliebte Version von Paperless) auf einem PC kann etwas technisch sein, aber mit den folgenden Schritten kannst du das problemlos schaffen. Paperless-ngx ist eine Open-Source-Software zur Verwaltung von Dokumenten, die lokal auf deinem PC läuft. 

---

### Voraussetzungen
1. **Betriebssystem**: Linux (Ubuntu/Debian empfohlen), macOS oder Windows (via WSL/Container).
2. **Software/Tools**: 
   - Docker und Docker Compose (empfohlen)
   - Alternativ: Python und eine manuelle Installation (komplizierter)

---

### **Installation mit Docker (empfohlen)**

1. **Docker installieren**
   - Linux: Installiere Docker mit den folgenden Befehlen:
     ```bash
     sudo apt update
     sudo apt install docker docker-compose
     ```
   - Windows/macOS: Lade Docker Desktop herunter und installiere es von [docker.com](https://www.docker.com/).

2. **Paperless-ngx-Dateien herunterladen**
   - Öffne ein Terminal und klone das Paperless-ngx-Repository:
     ```bash
     git clone https://github.com/paperless-ngx/paperless-ngx.git
     cd paperless-ngx
     ```

3. **Docker-Compose-Datei anpassen**
   - Es gibt eine Beispiel-Compose-Datei im Repository. Kopiere diese:
     ```bash
     cp docker-compose.yml.example docker-compose.yml
     ```
   - Bearbeite die Datei (z. B. mit `nano`) und passe die Umgebungsvariablen wie Speicherort der Dateien an:
     ```yaml
     volumes:
       - /path/to/data:/usr/src/paperless/data
       - /path/to/media:/usr/src/paperless/media
     ```
     Ersetze `/path/to/data` und `/path/to/media` durch tatsächliche Pfade auf deinem PC.

4. **Docker-Container starten**
   - Starte Paperless-ngx mit dem folgenden Befehl:
     ```bash
     docker-compose up -d
     ```
   - Die App läuft jetzt im Hintergrund. Du kannst den Status prüfen:
     ```bash
     docker-compose ps
     ```

5. **Paperless-ngx aufrufen**
   - Öffne deinen Browser und gehe zu: `http://localhost:8000`.
   - Standard-Login:
     - Benutzername: `admin`
     - Passwort: `admin`
   - Ändere das Passwort sofort nach dem ersten Login!

---

### **Optionale Einstellungen**
- **OCR für Dokumentensuche**: Paperless unterstützt die Texterkennung. Stelle sicher, dass die OCR-Pakete in deiner Umgebung installiert sind.
- **Backup**: Konfiguriere regelmäßige Backups für deine Daten und Medien.

---

### Alternative Installation (manuell ohne Docker)
Für fortgeschrittene Nutzer kannst du Paperless-ngx mit Python direkt installieren. Hier sind die groben Schritte:

1. Installiere Python 3.10+ und PostgreSQL.
2. Klone das Paperless-ngx-Repository.
3. Installiere Abhängigkeiten mit `pip`.
4. Konfiguriere die Datenbank und starte die Anwendung.

Dies ist jedoch komplexer, und die Docker-Installation wird dringend empfohlen.

---

1. **Windows-Version**: Stelle sicher, dass du Windows 10/11 (64-Bit) oder eine neuere Version nutzt.

---

1. Lade **Docker Desktop** von der offiziellen Website herunter: [Docker Desktop](https://www.docker.com/products/docker-desktop/).
2. Installiere Docker Desktop und aktiviere während der Installation die Option für die Unterstützung des Windows Subsystem for Linux (WSL 2).
3. Starte Docker und stelle sicher, dass es einwandfrei läuft.

---

### **Schritt 3: Paperless-ngx einrichten**
1. **Repository klonen**:
   - Öffne die Eingabeaufforderung oder PowerShell und gib den folgenden Befehl ein:
     ```bash
     git clone https://github.com/paperless-ngx/paperless-ngx.git
     ```
   - Falls du kein Git installiert hast, lade das Repository direkt als ZIP-Datei von [GitHub](https://github.com/paperless-ngx/paperless-ngx) herunter und entpacke es.

2. **Konfigurationsdateien erstellen**:
   - Wechsle in den heruntergeladenen Ordner:
     ```bash
     cd paperless-ngx
     ```
   - Erstelle die Datei `docker-compose.yml` mit der Standardkonfiguration:
     ```bash
     cp docker-compose.example.yml docker-compose.yml
     ```

3. **Umgebungsvariablen konfigurieren**:
   - Öffne die Datei `.env` (im selben Ordner wie `docker-compose.yml`) in einem Texteditor.
   - Passe die Einstellungen an (z. B. Datenbankname, Benutzername, Passwort).

---

### **Schritt 4: Paperless starten**
1. Starte die Dienste mit Docker Compose:
   ```bash
   docker-compose up -d
   ```
   - Dies lädt alle notwendigen Images und startet die Paperless-Umgebung.

2. **Zugriff auf Paperless**:
   - Öffne deinen Browser und gehe zu `http://localhost:8000`.
   - Du solltest jetzt die Paperless-ngx-Weboberfläche sehen.

---

### **Schritt 5: Erste Schritte**
1. Erstelle ein Benutzerkonto.
2. Beginne mit dem Hochladen und Organisieren deiner Dokumente.

---

### **Schritt 6: Automatisierung (optional)**
- Richte Ordnerüberwachung oder OCR-Funktionen ein, um hochgeladene Dokumente automatisch zu verarbeiten.

---

### **Fehlerbehebung**
- **Docker startet nicht**: Stelle sicher, dass WSL 2 und Virtualisierung im BIOS aktiviert sind.
- **Port-Konflikte**: Überprüfe, ob der Port `8000` bereits von einem anderen Programm verwendet wird, und ändere ihn in der Datei `docker-compose.yml`.

---

### **Alternativen**
Wenn dir Docker zu komplex erscheint, kannst du auch eine virtuelle Maschine (z. B. VirtualBox oder Hyper-V) mit Linux erstellen und Paperless-ngx dort installieren.

### 14. Assistent

Um **Paperless-ngx** auf **Unraid** zu installieren, musst du die richtigen Docker-Container einrichten. Hier ist eine Übersicht über die benötigten Docker-Container und die Einrichtung:

---

### **1. Notwendige Docker-Container**
Für eine vollständige Installation von **Paperless-ngx** benötigst du folgende Container:

1. **Paperless-ngx**:
   - Hauptanwendung für die Dokumentenverwaltung.
2. **Datenbank (PostgreSQL)**:
   - Wird von Paperless-ngx zur Speicherung von Metadaten genutzt.
3. **Redis**:
   - Dient als Cache für verbesserte Leistung.
4. **(Optional) Watchtower**:
   - Automatische Aktualisierung der Docker-Container.

---

### **2. Schritte zur Installation**

#### **A. Community Applications Plugin**
1. Stelle sicher, dass du das **Community Applications Plugin** in Unraid installiert hast:
   - Gehe zu `Apps` > `Install Plugins`.
   - Suche nach „Community Applications“ und installiere es.

#### **B. Paperless-ngx Container**
1. Öffne den **Community Applications App Store**.
2. Suche nach „Paperless-ngx“.
3. Wähle die Paperless-ngx-Vorlage (oft von Linuxserver.io oder einer vertrauenswürdigen Quelle).
4. Konfiguriere die Einstellungen:
   - **Datenpfade**:
     - `/config` → Stelle einen Speicherort für Konfigurationsdaten bereit (z. B. `/mnt/user/appdata/paperless-ngx`).
     - `/consume` → Ordner, in den neue Dokumente hochgeladen werden.
     - `/data` → Ordner für gescannte Dokumente.
   - **Port**: Standardmäßig `8000` (kann bei Bedarf geändert werden).

#### **C. PostgreSQL Container**
1. Suche nach einem PostgreSQL-Container im Community Applications Store.
2. Konfiguriere den Datenbankpfad:
   - **Datenpfad**: `/var/lib/postgresql/data` → Lokaler Ordner (z. B. `/mnt/user/appdata/postgres`).
   - **Umgebungsvariablen**:
     - `POSTGRES_USER` → Benutzername (z. B. `paperless`).
     - `POSTGRES_PASSWORD` → Sicheres Passwort.
     - `POSTGRES_DB` → Name der Datenbank (z. B. `paperless`).

#### **D. Redis Container**
1. Suche nach einem Redis-Container.
2. Standardkonfiguration ist ausreichend.

---

### **3. Docker-Netzwerk**
- Stelle sicher, dass alle Container im gleichen Docker-Netzwerk laufen, z. B. `bridge`.
- Konfiguriere das **Docker Compose Setup** so, dass Paperless auf die Datenbank und Redis zugreifen kann.

---

### **4. Konfiguration von Paperless**
1. Setze die Umgebungsvariablen in Paperless-ngx (siehe Docker-Einstellungen oder `.env`):
   - **Datenbank**:
     - `PAPERLESS_DBHOST`: Hostname des PostgreSQL-Containers.
     - `PAPERLESS_DBNAME`: Name der Datenbank (z. B. `paperless`).
     - `PAPERLESS_DBUSER`: Benutzername (z. B. `paperless`).
     - `PAPERLESS_DBPASS`: Passwort.
   - **Redis**:
     - `PAPERLESS_REDIS`: Hostname des Redis-Containers (z. B. `redis://redis:6379`).

2. Starte den Paperless-ngx-Container und überprüfe, ob er ordnungsgemäß läuft.

---

### **5. Testen**
- Rufe Paperless-ngx in deinem Browser auf: `http://<unraid-server-ip>:8000`.
- Erstelle einen Admin-Benutzer oder nutze die Umgebungsvariablen (`PAPERLESS_ADMIN_USER` und `PAPERLESS_ADMIN_PASSWORD`).

| `PAPERLESS_DBPASS`         | Passwort für die Datenbank      | `dein_sicheres_passwort`               |
| `PAPERLESS_REDIS`          | Hostname des Redis-Containers   | `redis://redis:6379`                   |
| `PAPERLESS_ADMIN_USER`     | Benutzername für Admin-Login    | `admin`                                |
| `PAPERLESS_ADMIN_PASSWORD` | Passwort für Admin-Login        | `dein_sicheres_admin_passwort`         |

3. Klicke auf **Add**, um jede Variable hinzuzufügen.

---

### **4. Speichere die Änderungen**
1. Scrolle nach unten und klicke auf **Apply**.
2. Unraid startet den Container neu, um die Änderungen zu übernehmen.

---

### **5. Prüfe die Änderungen**
1. Rufe die Paperless-Weboberfläche auf: `http://<unraid-server-ip>:8000`.
2. Melde dich mit dem **Admin-Benutzer** und dem in den Umgebungsvariablen festgelegten Passwort an.

---

### **6. (Optional) Überprüfe die Logs**
Falls Paperless nicht wie erwartet startet, überprüfe die Logs des Containers:
1. Klicke auf den **Log**-Button des Paperless-ngx Containers.
2. Suche nach Fehlermeldungen, die auf eine falsche Konfiguration hinweisen könnten.

---

### Hinweis
- Stelle sicher, dass der **PostgreSQL-Container** und der **Redis-Container** vor dem Paperless-ngx Container laufen.
- Wenn du die Hostnamen `postgres` oder `redis` für die Umgebungsvariablen nutzt, müssen alle Container im gleichen Docker-Netzwerk sein.
