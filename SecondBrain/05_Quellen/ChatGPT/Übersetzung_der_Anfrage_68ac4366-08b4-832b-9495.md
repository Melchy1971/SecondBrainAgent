Ziel klären
Du willst eine Datenbank für Promptverwaltung in Kategorien. Kernfunktionen, Kategorien, Tags, Versionen, Suche, Nutzung protokollieren, später erweiterbar.

Vorschlag Kernschema
• Tabellen, categories, tags, prompts, prompt_versions, prompt_tags, prompt_runs  
• Beziehungen, Prompt gehört zu einer Kategorie, Prompt hat viele Tags, Prompt hat viele Versionen, prompt_runs speichert Nutzung  
• Pflichtfelder in prompts, title, body, category_id, variables als JSON, language, model, status, created_at, updated_at  
• Optional, embedding Vektor für semantische Suche, Bewertung, Owner

SQL zum Start
Ich habe dir eine lauffähige PostgreSQL Schema-Datei erstellt. Enthält Tabellen, Indizes, Beispieldaten und Versionierung.
Download, sandbox:/mnt/data/prompt_db_schema_postgres.sql

Typische Kategorien
• Marketing, Produkt, Technik  
• Vertrieb, Support, HR  
• Forschung, Bildung, Recht

Namenskonventionen
• Titel kurz und eindeutig  
• Variablen im Prompt als {{variable}}  
• Status, draft, approved, deprecated  
• Versionen als ganze Zahl ab 1

Beispiele für Felder variables
• name, type, required, example  
• Beispiel, [{"name":"zielgruppe","type":"string","required":true},{"name":"thema","type":"string","required":true}]

Nützliche Abfragen
• Aktive Prompts einer Kategorie mit Tag
SELECT p.id, p.title FROM prompts p JOIN categories c ON c.id=p.category_id JOIN prompt_tags pt ON pt.prompt_id=p.id JOIN tags t ON t.id=pt.tag_id WHERE c.name='Marketing' AND t.name='LinkedIn' AND p.is_active=true ORDER BY p.updated_at DESC;
• Freitextsuche
SELECT id, title FROM prompts WHERE title ILIKE '%hook%' OR body ILIKE '%hook%' ORDER BY similarity(title, 'hook') DESC LIMIT 20;
• Neueste Version laden
SELECT pv.* FROM prompt_versions pv WHERE pv.prompt_id=$1 ORDER BY version DESC LIMIT 1;

Drei Umsetzungswege
1. Relationale DB, Postgres oder Supabase  
Vorteile, starke Abfragen, FTS, pgvector, Rechte. Nachteile, Setup und Pflege.
2. No-Code, Airtable oder Notion DB  
Vorteile, schnell, UI sofort da. Nachteile, schwächere Versionierung, begrenzte Abfragen.
3. Leichtgewicht, SQLite lokal  
Vorteile, simpel, portabel. Nachteile, kein Multi-User, kein Rechtemodell.

Best Practices
• Immer Versionen speichern  
• Variablen und Beispiele dokumentieren  
• Tags sparsam und konsistent  
• Indizes für Titel, Body, Embeddings  
• Nutzung messen, tokens, Kosten, Rating

Häufige Fehler
• Keine Versionierung  
• Variablen im Freitext mischen  
• Kategorien und Tags doppelt führen  
• Keine Indizes, Suche wird langsam

Nächste Schritte
• Datei importieren, Postgres anlegen  
• Kategorien und Tags anpassen  
• Bestehende Prompts importieren  
• Rollen festlegen, Owner pro Prompt
